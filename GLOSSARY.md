## SIEM & Security Terms

### SIEM (Security Information and Event Management)
A security solution that provides real-time analysis of security alerts generated by applications and network hardware. Combines Security Information Management (SIM) and Security Event Management (SEM).

### Alert
A notification generated when a detection rule matches specific log patterns or conditions. Alerts indicate potential security incidents requiring investigation.

### Detection Rule
A set of conditions and logic that identifies suspicious or malicious activity in log data. Rules can detect patterns like brute force attacks, unauthorized access, or anomalous behavior.

### Event
A single occurrence or action recorded in a log. Examples: user login, file access, network connection, system error.

### Correlation
The process of identifying relationships between multiple events across different log sources to detect complex attack patterns.

### False Positive
An alert triggered by benign activity that was incorrectly identified as malicious or suspicious.

### Incident
A confirmed security event or series of events that requires investigation and response.

### IOC (Indicator of Compromise)
Artifacts or observables that indicate a potential security breach, such as IP addresses, file hashes, or suspicious URLs.

### Threat Intelligence
Information about current or potential cyber threats used to enhance detection capabilities and inform security decisions.

### CVE (Common Vulnerabilities and Exposures)
A standardized identifier for known security vulnerabilities (e.g., CVE-2024-1234).

## SIEMBox Components

### Parser
A rule that transforms raw log messages into structured data with named fields. Parsers use regex patterns, grok patterns, or JSON parsing to extract information from logs.

**Example:**
```
Raw log: "192.168.1.100 - [2025-01-09] GET /api/users 200"
Parsed fields: {client_ip: "192.168.1.100", timestamp: "2025-01-09", method: "GET", path: "/api/users", status_code: "200"}
```

### Parser Engine
The service that applies parsers to raw logs in priority order, transforming unstructured text into structured data for analysis.

### Rules Engine
The service that evaluates parsed logs against detection rules to identify security events and generate alerts.

### Log Shipper
A lightweight agent that collects logs from various sources (files, Docker containers, journald) and forwards them to the SIEMBox backend via syslog.

### Ghost Shipper
A log shipper that continues sending logs with an invalid or expired API key, using cached configuration. Ghost shippers are visible in the UI for administrators to identify and fix.

### Syslog Server
The component that receives log messages via UDP or TCP on port 514, following RFC 3164/5424 syslog protocol standards.

### Asset Discovery
The automated process of scanning networks to identify hosts, services, and potential vulnerabilities using tools like NMAP.

## Log Management

### Raw Log
An unparsed log message as received from the log source, stored in the `raw_logs` table. Contains only the message portion after syslog protocol parsing.

### Parsed Log
A log that has been processed by the parser engine, with structured fields extracted and stored in the `parsed_logs` table as JSONB data.

### Log Source
The origin of log messages, such as web servers (Nginx), authentication systems (Authelia), DNS servers (Pi-hole), or applications.

### Syslog
A standard protocol for message logging (RFC 3164/5424). Consists of three parts:
- **Priority (PRI)**: Facility and severity
- **Header**: Timestamp, hostname, app name
- **Message**: The actual log content

**Example:** `<134>Jan 9 10:00:00 webserver nginx: GET /api 200`

### Facility
A syslog classification indicating the source type (e.g., `local0`, `auth`, `daemon`, `kern`).

### Severity
The importance level of a log message:
- **Emergency (0)**: System is unusable
- **Alert (1)**: Action must be taken immediately
- **Critical (2)**: Critical conditions
- **Error (3)**: Error conditions
- **Warning (4)**: Warning conditions
- **Notice (5)**: Normal but significant
- **Informational (6)**: Informational messages
- **Debug (7)**: Debug-level messages

### Log Retention
The policy defining how long logs are stored before automatic deletion. Configured per log type (raw logs, parsed logs, alerts).

### Priority
In parsers, the order in which patterns are evaluated. Lower numbers = higher priority. The first matching parser wins.

## Architecture & Technical

### Layered Architecture
A software design pattern where code is organized into layers with specific responsibilities:
- **Routes**: HTTP request handlers
- **Services**: Business logic
- **Models**: Data access
- **Database**: Storage

### Repository Pattern
A design pattern where database operations are encapsulated in model classes with static methods (e.g., `UserModel.findById()`).

### Microservices
An architectural style where an application is composed of small, independent services. SIEMBox uses 3 main services: Frontend, Backend, Database.

### REST API (Representational State Transfer)
An architectural style for building web APIs using HTTP methods (GET, POST, PUT, DELETE) and standard status codes.

### Middleware
Functions that execute between receiving a request and sending a response. Used for authentication, logging, error handling, etc.

### Hot Module Replacement (HMR)
A feature in Vite that updates modules in the browser without a full page reload, speeding up development.

### Single Page Application (SPA)
A web application that loads a single HTML page and dynamically updates content without full page reloads. SIEMBox frontend is an SPA built with Vue.js.

### Docker Container
A lightweight, standalone package that includes everything needed to run an application: code, runtime, libraries, and dependencies.

### Docker Compose
A tool for defining and running multi-container Docker applications using a YAML configuration file.

## Database & Storage

### PostgreSQL
An open-source relational database management system. SIEMBox uses PostgreSQL 15 for storing logs, alerts, users, and configuration.

### JSONB
A binary JSON data type in PostgreSQL that allows flexible schema storage while maintaining queryability. Used for parsed log fields and parser configurations.

**Example:**
```sql
SELECT * FROM parsed_logs WHERE parsed_data->>'status_code' = '404';
```

### Connection Pool
A cache of database connections that can be reused, improving performance by avoiding the overhead of creating new connections for each request.

### Foreign Key
A database constraint that ensures referential integrity by linking records in one table to records in another.

### Index
A database structure that improves query performance by creating a lookup table for specific columns.

### Migration
A versioned script that modifies database schema (adding tables, columns, indexes). SIEMBox uses SQL migration files.

### Transaction
A sequence of database operations that are treated as a single unit. Either all operations succeed (commit) or all fail (rollback).

### Parameterized Query
A SQL query that uses placeholders ($1, $2) instead of direct string concatenation, preventing SQL injection attacks.

**Safe:**
```typescript
query('SELECT * FROM users WHERE id = $1', [userId])
```

**Unsafe:**
```typescript
query(`SELECT * FROM users WHERE id = ${userId}`) // DON'T DO THIS
```

## Authentication & Authorization

### JWT (JSON Web Token)
A compact, URL-safe token format for securely transmitting information between parties. Used for API authentication in SIEMBox.

**Structure:** `header.payload.signature`

### Bearer Token
An authentication method where the token is sent in the HTTP `Authorization` header:
```
Authorization: Bearer <token>
```

### bcrypt
A password hashing algorithm that incorporates a salt and work factor to securely store passwords.

### Session
A server-side storage mechanism for maintaining user state across multiple requests. SIEMBox stores sessions in the database.

### Role-Based Access Control (RBAC)
A security model where permissions are assigned to roles, and users are assigned roles. SIEMBox roles:
- **Admin**: Full system access
- **Analyst**: Log and alert management
- **Viewer**: Read-only access
- **Operator**: Asset and scan management

### API Key
A secret token used by log shippers to authenticate with the SIEMBox API. 64-character hex strings stored securely in the database.

### Shipper ID
An 8-character identifier derived from the API key using SHA256 hashing, used to track which shipper sent each log.

### CORS (Cross-Origin Resource Sharing)
A security mechanism that controls which domains can access a web API from the browser.

## Development & Deployment

### TypeScript
A typed superset of JavaScript that compiles to plain JavaScript. Used throughout SIEMBox for type safety.

### Vue.js
A progressive JavaScript framework for building user interfaces. SIEMBox uses Vue 3 with the Composition API.

### Composition API
Vue 3's modern API for organizing component logic using functions like `ref()`, `computed()`, and `onMounted()`.

### Pinia
The official state management library for Vue.js, used in SIEMBox for managing application state.

### Express
A minimal and flexible Node.js web application framework used to build the SIEMBox backend API.

### Vite
A next-generation frontend build tool that provides fast development server startup and hot module replacement.

### Element Plus
A Vue 3 component library providing pre-built UI components like tables, forms, buttons, and dialogs.

### Winston
A logging library for Node.js that supports multiple transports (console, file) and structured logging.

### Jest
A JavaScript testing framework used for backend unit and integration tests.

### Vitest
A Vite-native testing framework used for frontend component tests.

### Environment Variable
Configuration values stored outside the codebase, typically in a `.env` file. Used for secrets, API URLs, and environment-specific settings.

**Example:**
```bash
DB_PASSWORD=secret123
JWT_SECRET=random-secret-key
```

### Hot Reload
Automatic reloading of code changes during development without manually restarting the server or refreshing the browser.

### Build
The process of compiling TypeScript to JavaScript and bundling assets for production deployment.

### Linting
Automated code analysis to identify potential errors, style violations, and anti-patterns. SIEMBox uses ESLint.

### Code Coverage
A metric showing what percentage of code is executed by tests. Higher coverage generally indicates better-tested code.

## Network & Protocol Terms

### UDP (User Datagram Protocol)
A connectionless network protocol used for fast, fire-and-forget communication. Syslog commonly uses UDP port 514.

### TCP (Transmission Control Protocol)
A connection-oriented protocol that guarantees delivery. Syslog can use TCP port 514 for reliable log transmission.

### Port
A numerical identifier for a specific service on a host. Common SIEMBox ports:
- **514**: Syslog (UDP/TCP)
- **3000**: Frontend (HTTP)
- **3001**: Backend API (HTTP)
- **5432**: PostgreSQL (internal)

### Reverse Proxy
A server that sits in front of web servers and forwards client requests. Nginx in SIEMBox acts as a reverse proxy for the API.

### Rate Limiting
A technique to control the number of requests a client can make to an API within a time window, preventing abuse.

### Webhook
An HTTP callback that sends real-time data to other applications when an event occurs.

## Parser & Rule Terms

### Regex (Regular Expression)
A sequence of characters that defines a search pattern. Used in parsers to extract structured data from log messages.

**Example:**
```regex
^\[(?<timestamp>[^\]]+)\].*(?<status>\d{3})
```

### Named Capture Group
A regex feature that assigns names to captured portions of text, making extracted data easier to reference.

**Syntax:** `(?<name>pattern)`

### Grok Pattern
A simplified pattern syntax for parsing logs, originally from Logstash. SIEMBox supports basic grok-to-regex conversion.

### Field Mapping
The association between parser capture groups and structured field names in parsed logs.

### Condition
A single check in a detection rule that compares a log field to a value using an operator (equals, contains, regex, etc.).

### Aggregation
A rule feature that counts or groups events over time to detect patterns like:
- **Threshold**: Trigger if count exceeds limit
- **Distinct Count**: Trigger if unique values exceed limit
- **Timeframe**: Window for counting events

**Example:** "5 failed logins from the same IP in 5 minutes"

### Severity
The importance level of an alert:
- **Low**: Informational, low risk
- **Medium**: Potential issue requiring investigation
- **High**: Serious issue requiring prompt action
- **Critical**: Severe threat requiring immediate response

## Security Testing & Scanning

### NMAP (Network Mapper)
An open-source network scanning tool used for host discovery, port scanning, service detection, and OS fingerprinting.

### Vulnerability Scan
An automated assessment that identifies security weaknesses in systems, such as outdated software, misconfigurations, or known CVEs.

### Port Scan
A technique to identify open network ports on a target system, revealing running services.

### Service Detection
The process of identifying what software and version is running on an open port (e.g., "Apache 2.4.52").

### Asset Inventory
A database of discovered network hosts, their services, and attributes, automatically maintained through periodic scans.
