# SIEMBox Documentation

This directory contains all project documentation for the SIEMBox SIEM system.

## Directory Structure

```
docs/
├── README.md                    # This file
└── project-management/          # Project planning and architecture docs
    └── architecture-diagram.md  # High-level system architecture
```

## Documentation Categories

### Project Management
- **Architecture Diagrams**: High-level system design and component relationships
- **Project Plans**: Development roadmaps and milestones
- **Requirements**: Functional and non-functional requirements

### Technical Documentation
- **API Documentation**: REST API specifications and examples
- **Deployment Guides**: Installation and configuration instructions
- **Development Guides**: Coding standards and best practices

### User Documentation
- **User Manuals**: End-user guides and tutorials
- **Administration Guides**: System administration and maintenance

## Contributing to Documentation

When adding new documentation:

1. Place files in the appropriate subdirectory
2. Follow the existing naming conventions
3. Include clear descriptions and examples
4. Update this README when adding new categories

## Architecture Reference

The main architecture diagram is located in `project-management/architecture-diagram.md`. This diagram serves as the primary reference for understanding the system's component relationships and data flow.

**Important**: Always refer to the architecture diagram when implementing new components or making architectural decisions to ensure consistency with the overall system design.
