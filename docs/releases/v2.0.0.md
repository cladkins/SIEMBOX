# SIEMBox v2.0.0 — The Parser Platform

SIEMBox v2 turns log onboarding and detection from a **code** problem into a
**data** problem. Parsers and detections are now portable, shareable, installable
from a community catalog, and generatable by AI — and a major normalization
overhaul means your detection rules actually fire.

## Highlights

### 🧩 Declarative parser engine
Parsers are pure data now — a match pattern, field mappings to a canonical
schema, and declarative `derivations` (`when` / `set` / `extract`). Onboard a new
log source without touching engine code. Every previously hardcoded per-parser
behavior moved into data and is applied generically.

### 📦 Community catalog — parsers AND detections, in-app
Browse and install **27 parsers** and **48 detection rules** straight from a
GitHub catalog (`cladkins/siembox-parsers`) — **Parsers → Browse Catalog** and
**Detection Rules → Browse Catalog**. Every item is validated and self-tested
before install; the dialog shows installed / update-available status, with search,
filters, and sorting. Export any parser as a portable `.parser.json`, or import one.

### 🤖 AI builder (parsers + detections)
Paste a log line → get a working parser. Describe a threat → get a detection rule.
The builder runs a **generate → validate → auto-refine loop** against the real
engine (up to 3 attempts), so it never hands you an invalid artifact. Bring your
own key — **Anthropic, OpenAI, or local Ollama** — in Settings → AI Builder (key
encrypted at rest, or via env var).

### 🎯 Detections that actually fire (canonical normalization)
A major normalization pass fixed the long-standing problem where rules silently
never matched because parsers spelled fields differently (`src_ip` vs `client_ip`
vs `source_ip`). Every parser now normalizes to one canonical schema, so a rule
written once matches logs from any source. Reversed `field_mappings` that silently
dropped IPs/status are fixed too.

### 🌍 GeoIP enrichment
Events are enriched with country / country-code / foreign-geo from an offline
DB-IP database (no external calls), powering geo-aware detections such as
foreign-login alerts.

### ➕ New parsers & rules
New first-party parsers for **Home Assistant, Plex, Jellyfin**, and **UniFi (CEF)**,
plus expanded detection coverage (cross-service auth, geo, media streaming,
password-manager, UniFi IDS/IPS).

### 🔌 Portable + community-ready
A documented `siembox.parser/v1` format, a strict validator + self-test runner,
and a CI gate (`validate-parsers` / `validate-detections`) so the community can
submit parsers and detections with confidence.

## Upgrading
```bash
git pull
docker compose -f compose.prod.yaml pull
docker compose -f compose.prod.yaml up -d
```
Database migrations run automatically on startup. New optional env vars (all with
safe defaults) — see `.env.example`:
- `CREDENTIAL_ENCRYPTION_KEY` — to store an AI key from the UI (encrypted at rest)
- `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` — AI builder keys (or set them in Settings)
- `SIEMBOX_CATALOG_REPO` / `SIEMBOX_CATALOG_REF` — point the catalog elsewhere
- `GITHUB_TOKEN` — raises the catalog's GitHub API rate limit

Bundled parsers/detections still seed on first run; the catalog is the additive
update path.

## Stabilization (fixed since the initial cut)
The known issues from the first v2.0.0 cut have been resolved:
- Vulnerability **dashboard** stat cards reading `0` — fixed (aligned the cards to
  the fields the assets/vulnerabilities APIs return, and made the charts re-render
  when stats load).
- Nuclei scan results on a CIDR **collapsing onto one asset** — fixed (assets are
  now keyed on each finding's matched host, and the `cve_id` column was widened so
  non-CVE findings persist instead of silently rolling back).
- Vulnerability-detail **dead / unsafe links** — fixed (references and the CVE link
  are only linkified for real http(s) URLs / real CVE ids; the asset link
  deep-links to the right host).
- **Dependency security sweep** — backend production image is down to a single
  no-upstream-fix moderate; both frontend criticals and every bundled-runtime high
  were cleared.
