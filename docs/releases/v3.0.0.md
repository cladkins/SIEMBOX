# SIEMBox v3.0.0 — The AI Security Analyst

SIEMBox v3 builds on the v2 Parser Platform with two big additions: a conversational
**AI Security Analyst** over your own data, and **SIEMBOX Endpoint agents** with
server-delivered YARA — plus an asset-correlation view, threat-intel upgrades, and
reliability fixes.

## Highlights

### 🤖 AI Security Analyst
A conversational, **read-only** SOC analyst embedded in SIEMBox. Ask about your alerts,
incidents, vulnerabilities, assets, and threat intel — and get help **prioritizing**.
It works through a model-agnostic, structured **tool loop** (one JSON action per turn),
so it runs on a **local Ollama** model or a cloud provider, with a **separate
analyst-model config** (Settings → AI Analyst) that can inherit the AI builder's. It's
grounded (every claim backed by a read-only tool), treats tool output as untrusted data,
and never executes model output. Available as a global **Ask AI** drawer and a dedicated
page, with **persisted per-user chat threads** and a transparent trace of which tools ran.

### 🖥️ SIEMBOX Endpoint agents + server-delivered YARA
Enroll lightweight **endpoint agents** from **Endpoints** with one-time enrollment
tokens. Agents report **inventory** (→ assets), **detections** (→ alerts, `source=edr`),
and **vulnerabilities** (→ asset vulnerabilities), and run **YARA** scans using rule packs
the server delivers. The starter bundle ships seeded; opt into a daily **YARA-Forge**
import (`EDR_YARA_FORGE_ENABLED`) or pull on demand, with server-side version retention
(`EDR_YARA_KEEP_VERSIONS`). The agent itself is a separate lightweight component.

### 🧭 Asset-360
Every asset now has a single correlation view tying together its open **vulnerabilities**,
the **alerts** it raised, its **endpoint agent** and **log shipper**, **GeoIP**, and open
**ports** — so an endpoint, a scanned host, and a log source resolve to one picture.

### 🌐 Threat Intelligence upgrades
External **blocklist feeds** (abuse.ch Feodo, Tor exit nodes, blocklist.de) and BYO-key
**reputation** (AbuseIPDB, **AlienVault OTX** — which replaced GreyNoise). The deprecated
abuse.ch **SSLBL** feed was removed. IP drill-down and an offline dashboard **country
choropleth** round out the investigation hub.

### 🤝 Community catalog contribution
Propose your parsers **and** detections to the community catalog directly from the UI via
a no-credential **browser hand-off** that opens a prefilled GitHub "new file" PR under your
own account.

### 🛡️ Reliability & correctness
- Fixed the post-deploy **black page** (the SPA now never caches `index.html` and
  self-heals stale chunk references after a redeploy).
- Added a **frontend type-check gate** (`vue-tsc`) to the image build so that class of
  runtime-only bug can't ship again.
- Clearer `CREDENTIAL_ENCRYPTION_KEY` errors + a startup check.
- The **Admin dashboard now reports the real version** (was hardcoded `0.1.0`).

## Upgrading
```bash
docker compose -f compose.prod.yaml pull
docker compose -f compose.prod.yaml up -d
# or pin the version:
SIEMBOX_VERSION=3.0.0 docker compose -f compose.prod.yaml up -d
```
Database migrations run automatically on startup and are idempotent. New optional env vars
(safe defaults) — see `.env.example`:
- `EDR_YARA_FORGE_ENABLED` / `EDR_YARA_KEEP_VERSIONS` — SIEMBOX Endpoint YARA delivery
- `SIEMBOX_CATALOG_DETECTIONS_PATH` — detections directory in the catalog repo

The AI Analyst is configured in **Settings → AI Analyst** (or it inherits **AI Builder**);
a stable 64-hex `CREDENTIAL_ENCRYPTION_KEY` is required to store provider keys at rest.

**Images:** `ghcr.io/cladkins/siembox-{backend,frontend,log-shipper}:3.0.0` (also `:3.0`, `:3`, `:latest`).
